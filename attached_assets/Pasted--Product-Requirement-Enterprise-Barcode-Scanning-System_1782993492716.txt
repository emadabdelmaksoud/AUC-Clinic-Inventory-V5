# Product Requirement: Enterprise Barcode & Scanning System

## Objective

Redesign the existing **Barcode & Labels** module into a complete **Barcode Management & Scanning System** that supports the entire inventory lifecycle rather than only barcode generation and printing.

The new module must follow enterprise inventory management best practices used in hospitals and warehouse management systems.

The implementation must preserve the current architecture, maintain backward compatibility, and integrate seamlessly with the existing Products, Assets, Inventory, Transactions, Audit Log, and PWA functionality.

---

# General Requirements

* Do NOT rewrite the application architecture.
* Preserve existing database structure whenever possible.
* Add only the minimum required schema changes.
* All new database changes must be provided as SQL migration scripts.
* Maintain compatibility with:

  * Supabase mode
  * Local Database mode
* Keep all current functionality operational.
* Follow existing UI/UX design language.

---

# Redesign the Barcode Module

The current module focuses on generating labels.

Instead, redesign it as a complete barcode management system.

Replace the current workflow with the following modules:

## 1. Barcode Assignment

This becomes the primary workflow.

Administrators should be able to:

* Assign a barcode to a Product
* Assign a barcode to an Asset
* Replace an existing barcode
* Remove a barcode
* Enter a barcode manually
* Generate a barcode automatically
* Scan an existing barcode using:

  * USB scanner
  * Camera scanner

Workflow:

Open Product/Asset

↓

Current Barcode

↓

Choose:

• Scan Barcode

• Generate Barcode

• Manual Entry

↓

Validate uniqueness

↓

Confirm

↓

Save

---

## 2. Barcode Printing

Printing must only occur after a barcode has been assigned.

Support:

* Single label
* Multiple labels
* Bulk printing
* Custom quantity
* PDF export
* PNG export
* SVG export
* Print preview

Allow administrators to manage templates.

Templates should support:

* Thermal labels
* Shelf labels
* Asset stickers
* QR labels
* Barcode labels

---

## 3. Scan & Verify

The scanner should NOT be a standalone page.

Instead, it should function as an input device throughout the application.

Create a dedicated verification mode.

When scanning:

The system should identify:

* Product
* Asset
* Unknown barcode
* Duplicate barcode

Display:

* Name
* Barcode
* Warehouse
* Current stock
* Asset status
* Current custodian

If the barcode is invalid, display a clear error.

# FIFO Enhancement

Whenever possible, barcode scanning should identify the batch automatically.

If the scanned barcode belongs to a specific batch:

* Product
* Batch
* Expiry

* Available quantity

# User Experience

The barcode system should feel like an enterprise Warehouse Management System.

Users should rarely search manually.

Instead, the standard workflow throughout the application should become:

**Scan → Identify → Perform Action → Save**

This scan-first approach should become the default interaction model for inventory operations.
