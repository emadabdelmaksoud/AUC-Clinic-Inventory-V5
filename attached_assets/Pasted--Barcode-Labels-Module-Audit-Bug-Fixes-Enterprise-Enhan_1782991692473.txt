# Barcode & Labels Module – Audit, Bug Fixes & Enterprise Enhancements

The Barcode & Labels module is functional but still lacks several enterprise features required for a professional hospital inventory management system.

## IMPORTANT REQUIREMENTS

* Do NOT rewrite the existing architecture.
* Do NOT change the database schema unless absolutely necessary.
* Preserve all existing functionality.
* Follow the current project coding style.
* Keep backward compatibility.
* If database changes are required, provide SQL migration scripts instead of modifying existing tables directly.
* Ensure all new features work in both the current Supabase deployment and the Local Database mode.

---

# Priority 1 – Fix Existing Bugs

## 1. Camera Scanner is not working

This is the highest priority.

The "Tap to open camera" button does not successfully open and scan barcodes/QR codes.

Investigate and fix:

* Camera permissions
* Browser compatibility
* HTTPS/PWA requirements
* Mobile support
* Desktop support
* Error handling when permission is denied
* Proper cleanup when camera closes
* Switching between front/back cameras on mobile
* Selecting different cameras when multiple devices exist

Requirements:

* Camera should automatically detect:

  * Code128
  * Code39
  * EAN-13
  * EAN-8
  * UPC
  * QR Codes

* Scanner should stop automatically after a successful scan.

* Add a loading indicator while initializing the camera.

* Display friendly error messages if:

  * Camera unavailable
  * Permission denied
  * No camera found
  * Browser unsupported

---

## 2. USB Barcode Scanner

Ensure USB scanners work automatically.

When the scanner page is open:

* Cursor automatically focuses on scan input.
* Scanner input is detected automatically.
* Enter key triggers search.
* No mouse interaction required.
