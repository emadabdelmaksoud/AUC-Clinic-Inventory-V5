Implement a complete User Profile module using the existing User Management as the single source of truth. Do not duplicate user information in the Custody module.

Requirements:

1. Create a User Profile page for every user that displays:

   * Full Name
   * Employee ID
   * Username
   * Email
   * Department
   * Position/Job Title
   * Phone Number
   * Role
   * Status (Active/Inactive)
   * Profile photo (optional)
   * Date Created
   * Last Login
   * Assigned Custody Assets
   * Custody History
   * Asset Transfer History
   * Maintenance History (if applicable)
   * Activity Log

2. The User Profile must always read its information directly from the User Management records. There must be only one source of truth for user information.

3. In the Custody module:

   * When creating a custody record, selecting a user should automatically populate all available user information (Employee ID, Department, Position, Email, etc.).
   * Store only the user's unique ID in the custody record as the relationship.
   * Display the user's information dynamically from the User table instead of duplicating it.

4. When a user's profile information is updated in User Management, all custody records and profile pages must immediately reflect the updated information without requiring manual edits.

5. The User Profile should automatically display:

   * Current assigned assets
   * Previous assets
   * Total assigned assets
   * Assignment dates
   * Return dates
   * Transfer history
   * Asset condition
   * Custody notes

6. Asset Transfers:

   * When transferring an asset, selecting the destination user should automatically load that user's information.
   * The asset should immediately disappear from the previous owner's active custody list and appear in the new owner's profile.
   * Preserve the complete transfer history with timestamps and users involved.

7. Navigation:

   * Add a "View Profile" action from User Management.
   * Add a clickable user name in Custody records that opens the user's profile.
   * Add a clickable custodian name in Asset Details that opens the user's profile.

8. Audit:

   * Log every custody assignment, transfer, return, and profile update.
   * Display these logs in the user's profile timeline.

9. Keep all existing business logic, database schema (except required relationships), authentication, permissions, and UI intact. Implement this as an enhancement without breaking existing functionality.
